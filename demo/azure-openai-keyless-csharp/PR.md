## Migrate from Azure OpenAI Chat Completions to Responses API (.NET)

### What this PR does

Migrates this .NET app from the Azure OpenAI SDK (`Azure.AI.OpenAI`) with Chat Completions to the OpenAI SDK (`OpenAI`) with the Responses API.

### Changes

| File | Change |
|---|---|
| `Program.cs` | `AzureOpenAIClient` / `GetChatClient` → `ResponsesClient` with `BearerTokenPolicy`; `CompleteChat(messages)` → `CreateResponseAsync(model, messages)`; `completion.Content[0].Text` → `response.GetOutputText()` |
| `*.csproj` | Removed `Azure.AI.OpenAI`, added `OpenAI` >= 2.9.1 |
| `infra/main.bicep` | Removed `openAiApiVersion` parameter and `OPENAI_API_VERSION` output |

### Migration details

- **Client construction**: `AzureOpenAIClient` + `new DefaultAzureCredential()` → `ResponsesClient` with `BearerTokenPolicy(credential, scope)` via `OpenAIClientOptions`
- **Endpoint**: Direct URI → `$"{endpoint}/openai/v1/"` (stable, no `api_version` querystring)
- **API call**: `chatClient.CompleteChat(messages)` → `responsesClient.CreateResponseAsync(model, messages)`
- **Message format**: `SystemChatMessage` / `UserChatMessage` → `ResponseItem.CreateSystemMessageItem()` / `ResponseItem.CreateUserMessageItem()`
- **Response shape**: `completion.Content[0].Text` → `response.GetOutputText()`
- **Pragma**: Added `#pragma warning disable OPENAI001` (Responses API is marked experimental)
- **Infra**: Removed `openAiApiVersion` parameter from Bicep — the `/openai/v1/` endpoint doesn't need it

### Why

GPT-5 and newer models require the Responses API. The `/openai/v1/` endpoint is stable and requires no `api_version` management.

### How to verify

```bash
dotnet build
dotnet run
```

The app should print a haiku about a hungry cat, same as before.
